Polydelve lets users bet on whether a CVE will be published for a top npm or PyPI package. Markets resolve automatically when OSV data confirms a vulnerability.

## What it does

- **Prediction markets** — buy contracts on packages you believe are at risk. Payout if a CVE lands before expiry.
- **CVE tracking** — monitors the top 500 npm and PyPI packages against the OSV advisory database.
- **EPSS trend charts** — exploitation probability over time with CVE scatter overlay.
- **Exploit signals** — PoC flags, active exploit flags, and OSV malicious advisory detection.
- **Leaderboard** — users ranked by prediction accuracy over resolved markets.
- **Security news** — curated feed of security events tagged by package and sector.

## Core concepts

| Concept | Description |
|---------|-------------|
| **Contract** | A bet that a specific package will receive a CVE above a CVSS threshold before a deadline. |
| **Schmeckles** | In-platform currency. New users start with 1000. |
| **EPSS** | Exploit Prediction Scoring System — probability a CVE gets exploited in the wild. |
| **OSV** | Open Source Vulnerability database. The source of truth for CVE resolution. |
| **Market** | A featured contract displayed on the home feed, curated for high-interest packages. |
