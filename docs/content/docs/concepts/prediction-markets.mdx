## What is a contract?

A contract is a bet that a target package will receive a CVE meeting specific criteria before a deadline. Users buy contracts for a fixed price; if the condition is met, they receive a payout.

### Contract parameters

| Field | Description |
|-------|-------------|
| `package_name` | Target npm or PyPI package |
| `ecosystem` | `npm` or `PyPI` |
| `cvss_threshold` | Minimum CVSS score to trigger resolution (e.g. 7.0) |
| `epss_threshold` | Minimum EPSS score to trigger resolution (optional) |
| `purchase_price` | Cost in Schmeckles |
| `duration_days` | Days until the contract expires unresolved |

### Resolution

Contracts resolve **YES** when OSV records a new CVE against the package that meets the CVSS/EPSS thresholds before the deadline. Resolution runs on the EPSS refresh cycle.

Contracts expire **NO** if the deadline passes without a qualifying CVE.

## Pricing

Contract price is calculated in `features/contract_pricing.py`. Inputs:

- **Base EPSS score** — higher exploitation probability → higher cost
- **CVE history weight** — packages with frequent past CVEs cost more
- **Download weight** — widely-used packages carry higher demand premium
- **Multiplier** — payout = `purchase_price × multiplier`

The multiplier is inversely proportional to probability: low-risk packages pay more per Schmeckle.

## Schmeckles

Users start with 1000 Schmeckles. Buying a contract deducts `purchase_price` immediately. Payouts credit on resolution. Balance is tracked in the `users` table.

There is no mechanism to earn Schmeckles outside of winning contracts — scarcity is intentional.

## User accounts

Users are created on first `GET /users/me` call after Auth0 login. New rows get `username = NULL`; the frontend blocks access until a username is chosen.

### Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/users/me` | Returns or creates the current user |
| `PATCH` | `/users/me` | Update `username` and/or `avatar_url` |
| `GET` | `/users/me/avatar-upload-url` | Returns a presigned S3 PUT URL + final public URL |
| `GET` | `/users/leaderboard` | Paginated leaderboard ranked by Schmeckles |

### Avatar uploads

`GET /users/me/avatar-upload-url?content_type=image/jpeg` returns `{ upload_url, public_url }`. The client PUTs the file directly to S3 using `upload_url`, then calls `PATCH /users/me` with `{ avatar_url: public_url }`. Avatars land in `s3://polydelve-avatars/avatars/{user_id}/{uuid}.{ext}` and are publicly readable. Write access is restricted to the ECS task role.

## Featured contracts

A **featured contract** is auto-generated by `features/featured_contracts.py` after each news ingest. The pipeline scores packages by `relevancy_score` (from GPT news analysis), EPSS, and CVE history, then creates contracts for the highest-signal packages. Featured contracts are reranked on each run and surfaced via `GET /featured-contracts`.
